 48  """ 
 49  annotationdata is a free and open source Python library to access and 
 50  search data from annotated data. It can convert file formats like Elan’s EAF, 
 51  Praat's TextGrid and others into a Transcription() object and convert into 
 52  any of these formats. Those objects allow unified access to linguistic data 
 53  from a wide range sources. 
 54   
 55  Details can be found in the following publication: 
 56   
 57      Brigitte Bigi, Tatsuya Watanabe, Laurent Prévot (2014). 
 58      Representing Multimodal Linguistics Annotated data, 
 59      Proceedings of the 9th edition of the Language Resources and Evaluation 
 60      Conference, 26-31 May 2014, Reykjavik, Iceland. 
 61   
 62  Linguistics annotation, especially when dealing with multiple domains, 
 63  makes use of different tools within a given project. 
 64  Many tools and frameworks are available for handling rich media data. 
 65  The heterogeneity of such annotations has been recognised as a key problem 
 66  limiting the inter-operability and re-usability of NLP tools and linguistic 
 67  data collections. 
 68   
 69  In annotation tools, annotated data are mainly represented in the form of 
 70  "tiers" or "tracks" of annotations. 
 71  The genericity and flexibility of "tiers" is appropriate to represent any 
 72  multimodal annotated data because it simply maps the annotations on the 
 73  timeline. 
 74  In most tools, tiers are series of intervals defined by: 
 75   
 76      * a time point to represent the beginning of the interval; 
 77      * a time point to represent the end of the interval; 
 78      * a label to represent the annotation itself. 
 79   
 80  In Praat, tiers can be represented by a time point and a label (such 
 81  tiers are named PointTiers and IntervalTiers). 
 82  Of course, depending on the annotation tool, the internal data representation 
 83  and the file formats are different. 
 84  For example, in Elan, unlabelled intervals are not represented nor saved. 
 85  On the contrary, in Praat, tiers are made of a succession of consecutive 
 86  intervals (labelled or un-labelled). 
 87   
 88  The annotationdata API used for data representation is based on the 
 89  common set of information all tool are currently sharing. 
 90  This allows to manipulate all data in the same way, regardless of the file 
 91  type. 
 92   
 93  This API supports to merge data and annotation from a wide range of 
 94  heterogeneous data sources for further analysis. 
 95   
 96  To get the list of extensions currently supported for reading and writing: 
 97   
 98      >>> ext = annotationdata.io.extensions 
 99
