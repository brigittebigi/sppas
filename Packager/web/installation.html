Install SPPAS

SPPAS is ready to run, so it does not need elaborate installation. All you need to do is to copy the SPPAS package from the website to somewhere on your computer.

The SPPAS package is compressed and zipped, so you will need to decompress and unpack it once you've got it. Choose preferably a location with only US-ASCII characters in the name of the path.

And... that's it! You installed SPPAS!

In order to work, external programs must be installed. Follow carefully the instructions of this page to download and install all of them: Python 2.7, wxPython 3.0, Julius.

Install external programs

Other programs are required for SPPAS to operate. Of course, they must be installed before using SPPAS, and only once. This operation takes 5 up to 10 minutes depending on the operating system.

Administrator rights are required to perform these installations.


Windows

  1. Click the link to download Python 2.7 "Windows x86 MSI installer", then execute it. Notice that even if your system is 64 bits, you have to install the 32 bits version of python.
  2. Click the link to download wxPython with version name: "-win32 -py27" (i.e. for windows 32-bits, and python 2.7); then execute it.
  3. Click the link to download Julius (any release >= 4.1). Unpack the zip file then go into the directory. Open the "bin" folder then select the file "julius.exe. Copy this file into C:\WINDOWS\. Notice that the file name must be strictly "julius.exe", i.e. without version number. If the ".exe" can't be seen in your Explorer, check the right option to show the extensions!


MacOS

  1. Click the link to download Python 2.7 "Mac OS X 64-bit/32-bit installer", then execute it.
    • Try this: download and install the following snapshot package: wxPython 3.0.3 cocoa
    • Or try this: Click the link to download wxPython 3.0.x stable package with version name: -cocoa and -py2.7.. Click on the package and follow instructions.
      • Notice that wxPython installer is not signed by Apple, so by default your Mac will tell you it is corrupted and will not install it. You can get around this by temporarily relaxing the security settings in System Preferences. In the Security & Privacy section set "Allow applications downloaded from" to "Anywhere."
    • For Julius software, I prepared a 64-bits version which include a ready-to-use (old) version of Julius for MacOS. This version is available here: Julius for MacOS. Download, extract the archive and drag-and-drop the package content into the /usr/local/bin directory.


Linux

  • There are deb packages for all dependencies (python2.7, wxpython, julius). python-wxgtk-media and gstreamer must be installed too.
  • There are rpm packages for python2.7 and wxpython. The Julius tool will have to be installed from sources.
