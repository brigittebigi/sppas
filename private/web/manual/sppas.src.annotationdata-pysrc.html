 40  @organization: Laboratoire Parole et Langage, Aix-en-Provence, France 
 41  @contact:      brigitte.bigi@gmail.com

 44  @summary:      Representation of annotated data. 
 45   
 46  annotationdata is a free and open source Python library to access and 
 47  search data from annotated data. It can convert file formats like Elan’s EAF, 
 48  Praat's TextGrid and others into a Transcription() object and convert into 
 49  any of these formats. Those objects allow unified access to linguistic data 
 50  from a wide range sources. 
 51   
 52  Details can be found in the following publication: 
 53   
 54      Brigitte Bigi, Tatsuya Watanabe, Laurent Prévot (2014). 
 55      Representing Multimodal Linguistics Annotated data, 
 56      Proceedings of the 9th edition of the Language Resources and Evaluation 
 57      Conference, 26-31 May 2014, Reykjavik, Iceland. 
 58   
 59  Linguistics annotation, especially when dealing with multiple domains, 
 60  makes use of different tools within a given project. 
 61  Many tools and frameworks are available for handling rich media data. 
 62  The heterogeneity of such annotations has been recognised as a key problem 
 63  limiting the inter-operability and re-usability of NLP tools and linguistic 
 64  data collections. 
 65   
 66  In annotation tools, annotated data are mainly represented in the form of 
 67  "tiers" or "tracks" of annotations. 
 68  The genericity and flexibility of "tiers" is appropriate to represent any 
 69  multimodal annotated data because it simply maps the annotations on the 
 70  timeline. 
 71  In most tools, tiers are series of intervals defined by: 
 72   
 73      * a time point to represent the beginning of the interval; 
 74      * a time point to represent the end of the interval; 
 75      * a label to represent the annotation itself. 
 76   
 77  In Praat, tiers can be represented by a time point and a label (such 
 78  tiers are named PointTiers and IntervalTiers). 
 79  Of course, depending on the annotation tool, the internal data representation 
 80  and the file formats are different. 
 81  For example, in Elan, unlabelled intervals are not represented nor saved. 
 82  On the contrary, in Praat, tiers are made of a succession of consecutive 
 83  intervals (labelled or un-labelled). 
 84   
 85  The annotationdata API used for data representation is based on the 
 86  common set of information all tool are currently sharing. 
 87  This allows to manipulate all data in the same way, regardless of the file 
 88  type. 
 89   
 90  This API supports to merge data and annotation from a wide range of 
 91  heterogeneous data sources for further analysis. 
 92   
 93  To get the list of extensions currently supported for reading and writing: 
 94   
 95      >>> ext = annotationdata.io.extensions
